# update_history.json 仕様書

`update_history.json` は、ポータルサイトなどで表示するゲームの更新履歴を管理するためのファイルです。

## ファイル形式
JSON形式（配列）

## データ構造
各要素は以下のフィールドを持つオブジェクトです。

| フィールド名 | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `version` | string | ○ | バージョン番号（例: "1.0", "1.1"） |
| `date` | string | ○ | 更新日（YYYY-MM-DD形式） |
| `title` | string | ○ | 更新内容のタイトル |
| `description` | string | ○ | 更新内容の概要説明 |
| `changes` | string[] | ○ | 具体的な変更項目のリスト（配列） |

## 記載ルール
1. **ユーザー視点**: 開発者向けのリファクタリング内容ではなく、プレイヤーにとってのメリットを記載する。
2. **マイナーバージョンまで**: バージョン表記はパッチバージョンを含めない "X.X" 形式を基本とする。
3. **最新順（降順）**: 新しい更新内容を配列の先頭（インデックス 0）に追加する。ポータル表示等でソート処理を介さず最新を表示するための必須ルールとする。

## サンプル
```json
[
  {
    "version": "1.0",
    "date": "2026-05-10",
    "title": "パフォーマンス改善",
    "description": "レンダリングエンジンの最適化により、動作がよりスムーズになりました。",
    "changes": [
      "描画処理のメモリ使用量を15%削減",
      "特定の条件下で発生していたグリッチの修正",
      "UIのレスポンス向上"
    ]
  }
]
```
