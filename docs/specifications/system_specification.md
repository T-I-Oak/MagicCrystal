# Magic Crystal ゲーム論理仕様書 (System Specification)

本ドキュメントは、実装方法を問わず「Magic Crystal」というゲームを成立させるための論理的な基盤仕様を定義したものです。

## 1. 座標系と構成

### 1.1 グリッド (Grid)
- **画面解像度**: 960 x 660 ピクセル (24 x 13 タイル + Header/Footer)
- **タイルサイズ**: 40 x 40 ピクセル
を 4×4 の論理ステップ（内部座標）として管理する。
- **内部解像度**: 横96ステップ × 縦52ステップ。
- **ステージ外の扱い**: 定義されたグリッド範囲外（$x < 0, x \ge 24, y < 0, y \ge 13$）へのアクセスは、一律で「岩 (ID 2)」として扱われる。これにより、画面外への脱走は不可能であり、ジャンプ中の衝突判定も画面端で発生する。

### 1.2 キャラクター座標
- キャラクターの座標 $(x, y)$ は論理ステップ単位で管理される。
- **整列状態 (Aligned)**: $x$ および $y$ が 4 で割り切れる状態（タイルの境界に位置している状態）。
- 収集判定や方向転換などの主要ロジックは、原則としてこの「整列状態」の時にのみ実行される。

## 2. タイル定義 (Tile IDs)

| ID | 名称 (論理名) | 特性 | 説明 |
|:---|:---|:---|:---|
| 0 | 背景 (Empty) | 通行可能 | 何もない空間。 |
| 1 | 土 (Dirt) | 進入不可 | 掘ることが可能な壁。物理的にキャラクターを遮る。 |
| 2 | 岩 (Rock) | 進入不可 | 掘ることができない、破壊不能な壁。ステージ境界もこの属性を持つ。 |
| 3 | ポータル (Portal) | 特殊 | スタート地点であり、かつ出口。クリスタル全回収後に活性化する。 |
| 4 | 紅晶 (Red Crystal) | 収集可能 | 回収すると「地震」カウントダウン(60F)を開始し、「土の粉塵」へ変化。 |
| 5 | 蒼晶 (Blue Crystal) | 収集可能 | 収集必須項目。回収すると地震カウントダウンを停止（クリア）し、「岩の粉塵」へ変化。 |
| 6 | 土の粉塵 (Dirt Trace)| 通行可能 | 穴を掘った跡、または紅晶の回収跡。地震発生時に「土」へ戻る。 |
| 7 | 岩の粉塵 (Rock Trace)| 通行可能 | 蒼晶の回収跡。地震発生時に「岩」へ戻る。 |

## 3. アクションと物理ロジック

全ての移動・アクションは、1フレーム（1チック）ごとのステップ更新として処理される。

### 3.1 通常移動 (歩行)
1. **入力受付**: 「整列状態」の時にのみ入力を受け付ける。
2. **振り向き**: 
   - **通常移動（左右キー/斜めジャンプ）**: 逆方向への入力があった場合、3フレームの待機時間（硬直）が発生する。誤操作を防止するための仕様。
   - **スマート操作（Q/Eキー）**: 
     - 入力した方向が現在の向きと逆なら **向きを変える（1フレーム、移動なし）**。
     - 入力した方向が現在の向きと同じなら **即座にジャンプ移動を開始する**。
     - 移動ボタンを押したそのフレームから座標が変化するため、振り向きからの土登頂は **最短9フレーム**（振り向き1F ＋ 移動計8F）で完了する。
3. **進行**: 進行方向に壁（ID 1, 2）またはステージ境界がある場合は停止する。
4. **摩擦/慣性**: 存在しない。入力があれば即座に等速移動（1ステップ/F）する。

### 3.2 穴掘り (Dig)
- **発動条件**: キャラクターが「整列状態」であり、足元（$y+1$ タイル）が「土 (ID 1)」であること。
- **効果**: 足元のタイルが「土の粉塵 (ID 6)」に変化する。
- **硬直**: 発動後、2フレームの待機時間（`turnWait = 2`）が発生し、その間アクションや移動は制限される。

### 3.3 ジャンプ (Jump)
ジャンプ中の移動方向 $(sx, sy)$ は、1フレームごとに「整列状態」のタイミングで以下の優先順位で決定される（斜めジャンプの場合）：

1. **水平判定**: 
   - 進行方向（左または右）にある隣接タイル $(gridX + dir, gridY)$ が壁（ID 1, 2 / 画面外含む）なら、水平移動を不可とし $sx = 0$ とする。
   - 壁がなければ $sx = dir$。
2. **垂直判定**: 
   - 上昇中または下降方向 $(vDir)$ にある隣接タイル $(gridX, gridY + vDir)$ が壁なら $sy = 0$。
   - 壁がなければ $sy = vDir$。
   - ただし、上昇中に画面上端 ($y \le 0$) に達した場合も $sy = 0$ となる。
3. **対角判定 (斜め衝突防止)**: 
   - $sx \neq 0$ かつ $sy \neq 0$ （斜めに進もうとしている）場合、移動先となる対角のタイル $(gridX + sx, gridY + sy)$ を確認する。
   - もし対角タイルが壁であれば、**垂直移動を優先し、水平移動をキャンセルする ($sx = 0$)**。
   - これにより、ブロックの角に引っかかることなく垂直に上昇・下降することが保証される。

ジャンプのフェーズ：
1. **開始**: 入力（上/斜め上）により `jumpState = 9` となる。
2. **上昇フェーズ**: $jumpState$ が 9 から 2 へ減少するまで（計8ステップ/2タイル分）。上記判定に基づき $(sx, sy)$ を計算して移動。
3. **下降フェーズ**: $jumpState = 1$ の状態。垂直判定 $vDir$ が「下（+y）」として扱われる。
4. **着地**: 移動後の「整列状態」において、足元（$y+1$）が壁であることを検知した瞬間に `jumpState = 0` となり、通常状態に戻る。

## 4. 地震 (アースクエイク) システム

### 4.1 カウントダウン
- 「紅晶 (ID 4)」を回収した瞬間、内部タイマーが **60フレーム** にセットされる。
- 1フレームごとにタイマーが1減少し、0になった瞬間に地震が発動（地形変異）する。

### 4.2 停止（クリア）
- タイマー作動中に「蒼晶 (ID 5)」を回収すると、タイマーは 0 にリセットされ、地震は未発生のまま終了する。

### 4.3 地形変異
地震発動時、ステージ内の全タイルをスキャンし以下の置き換えを行う：
- **ID 6 (土の粉塵)** $\to$ **ID 1 (土)**
- **ID 7 (岩の粉塵)** $\to$ **ID 2 (岩)**

### 4.4 埋没判定
- 地形変異の瞬間、プレイヤーの現在座標（整列状態での格子位置）にあるタイルが ID 1 または 2 になった場合、即座に「ミス（埋没）」となる。

## 5. ゲームサイクル
1. **スタート**: ステージ開始時、プレイヤーは「ポータル (ID 3)」の中心（整列された格子位置）に配置される。
2. **攻略**:
   - 画面内の「蒼晶」および「紅晶」を全収集することを目指す。
   - 「紅晶」による地震の発生を、「蒼晶」の取得順序や穴掘りによる退路確保で制御する。
3. **クリア基準**:
   - クリスタル残数が 0 の状態でポータルに触れる。
