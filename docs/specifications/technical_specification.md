# Magic Crystal 技術実装仕様書 (Technical Specification)

本ドキュメントは、JavaScript および HTML5 Canvas を用いた現在の「Magic Crystal」の実装詳細をまとめたものです。

## 1. 描画と解像度

### 1.1 ビューポート
- **キャンバスサイズ**: 横 960px × 縦 520px。
- **タイルサイズ**: 40px × 40px。
- **スケーリング**: `main.js` の `scaleGame` ロジックにより、ウィンドウサイズに合わせてアスペクト比を維持しつつ自動拡大縮小される。

### 1.2 レンダリングスタック
- **Renderer.js**: 毎フレーム `requestAnimationFrame` に同期して描画を実行。
- **カメラ**: 現在は固定画面（シングルスクリーン）方式。

## 2. アセット構成

### 2.1 タイルセット (`assets/tiles.png`)
4列 × 3行（1枚 40px）のスプライトシート。
- **1行目**: 背景(0), 土(1), 岩(2), ポータル(3)
- **2行目**: 紅晶(4), 蒼晶(5), 土粉塵(6), 岩粉塵(7)
- **3行目**: エディタカーソル(8)

### 2.2 プレイヤー (`assets/player.png`)
3列 × 3行のスプライトシート。
- **1行目**: 待機(0), 走り1(1), 走り2(2)
- **2行目**: ジャンプ(3), 落下(4), ミス(5)
- **3行目**: 穴掘り(6), 地震予兆/ライフ(7), クリア(8)

## 3. プログラム構造

### 3.1 クラス構成
- **Game.js**: 全体のステート管理（状態遷移、セーブ/ロード、更新ループ）。
- **Physics.js**: 衝突判定と移動ロジック。
- **Level.js**: ステージデータ（二次元配列）の保持と変異ロジック。
- **Input.js**: キーボードおよび外部入力を論理アクションに抽象化。バッファリングとエッジ検出をサポート（詳細は[入力システム仕様書](./input_specification.md)を参照）。
- **Assets.js**: 画像リソースのプリロードと分割管理。

### 3.2 入力マッピング
詳細は[入力システム仕様書](./input_specification.md)を参照してください。

## 4. データ保存
- **localStorage**: 設定情報（音量、操作感）およびステージ進捗を `JSON` 形式で保存。

## 5. 拡張性
- **LevelEditor**: 開発者モード（または将来的なユーザー作成モード）として、リアルタイムにタイル配置が可能なエディターを内蔵。
